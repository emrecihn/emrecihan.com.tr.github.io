Sistem Mimarisi

Windows Server Event ID 1053 Kesin Çözümü

Servislerin zamanında başlamaması sonucu oluşan timeout hatalarının Registry ile kalıcı olarak giderilmesi.

Windows Server ortamlarında, özellikle ağır ERP servisleri veya özel yazılımlar çalıştırırken en sık karşılaşılan sorunlardan biri Event ID 1053 hatasıdır. Sistem, servisin 30 saniye içinde yanıt vermesini bekler, vermezse servisi durdurur.

Çözüm Adımları:

  • Regedit (Kayıt Defteri Düzenleyicisi) açın.
  • HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control yoluna gidin.
  • Sağ tıklayıp yeni bir DWORD (32-bit) Value oluşturun ve adını ServicesPipeTimeout yapın.
  • Değer verisini Decimal (Ondalık) seçip 60000 (60 saniye) veya 120000 (120 saniye) yapın.
  • Sunucuyu yeniden başlatın.

Bu müdahale, Mikro API gibi geç yüklenen servislerin hata vermeden ayağa kalkmasını sağlar.

Ağ Güvenliği

WatchGuard IPS ve SQL Injection False-Positive Yönetimi

Firewall'un yasal SQL trafiğini saldırı (Signature 1058077) olarak algılamasını engelleme yöntemleri.

Kurumsal ağlarda WatchGuard gibi UTM cihazları kullanırken, içerideki özel yazılımların veritabanına attığı kompleks SQL sorguları, IPS (Intrusion Prevention System) tarafından SQL Injection saldırısı olarak yorumlanabilir.

Özellikle Signature ID: 1058077 alarmı alıyorsanız, kaynağın kendi iç sunucunuz olup olmadığını kontrol etmelisiniz. Eğer trafik güvenliyse, WatchGuard Policy Manager üzerinden IPS ayarlarına girip bu spesifik ID için bir Exception (İstisna) kuralı yazarak ağ akışını kesintisiz hale getirebilirsiniz.

Sanallaştırma

Proxmox VE Üzerinde Nginx Proxy Manager (NPM) Mimarisi

LXC container kullanarak local ağdaki servisleri SSL sertifikası ile dış dünyaya açmak.

Proxmox sanallaştırma ortamında kaynak tüketimini minimumda tutmak için sanal makineler (VM) yerine Linux Containers (LXC) kullanmak en verimli yoldur.

Neden Nginx Proxy Manager?

Ağınızda çalışan ERP arayüzü, IoT dashboard'ları veya şirket içi web sitelerini tek bir 80/443 portu üzerinden, üstelik otomatik Let's Encrypt SSL sertifikaları alarak dışarı açmanızı sağlar. Docker üzerinden bir LXC içine kurularak saniyeler içinde ayağa kaldırılabilir.

Veritabanı Yönetimi

Uzak SQL Server Erişimi İçin Güvenli Yapılandırma

SQL Server'ı dışarıdan erişime açarken alınması gereken güvenlik ve yetkilendirme tedbirleri.

Bir SQL Server'ı dış dünyadan erişilebilir hale getirmek büyük bir güvenlik riskidir. Sadece Port 1433'ü modeme yönlendirmek yeterli değildir.

Best Practice Uygulamaları:

  • Port Değişimi: Varsayılan 1433 portunu, tahmin edilemez (örn: 54932) bir porta taşıyın.
  • SA Hesabını Kapatın: `sa` hesabı bruteforce saldırılarının ilk hedefidir. Bu hesabı devre dışı bırakıp özel yetkili bir hesap oluşturun.
  • IP Kısıtlaması: Firewall üzerinden SQL portuna erişimi sadece bilinen statik IP adreslerine izin verecek şekilde (Whitelist) kısıtlayın.
Sistem Yönetimi

Active Directory Çoklu Şube Optimizasyonu

Active Directory Sites and Services ile farklı lokasyonlar arası replikasyon trafiğini düzenleme.

Birden fazla fabrikası veya ofisi olan kurumlarda Active Directory yapısı düzgün kurulmazsa, gereksiz ağ trafiği hatları tıkayabilir. Sites and Services menüsü üzerinden her şubenin Subnet'ini belirlemek, kullanıcıların kendi lokasyonlarındaki Domain Controller (DC) üzerinden oturum açmasını sağlar. Bu sayede WAN hatları gereksiz kimlik doğrulama paketleriyle meşgul edilmez.

ERP Entegrasyon

Mikro ERP Masaüstü API ve SQL Bağlantı Sorunları

Mikro Desktop API servisinin çalışmaması ve Timeout hatalarının giderilmesi.

Mikro ERP kullanan işletmelerde, dış yazılımların (B2B, CRM) ERP ile haberleşmesini sağlayan Masaüstü API servisleri bazen SQL Server ile olan bağlantı kopmaları yüzünden kilitlenebilir.

Çözüm için öncelikle Windows Hizmetleri (Services.msc) üzerinden API servisinin "Kurtarma" (Recovery) sekmesine gidip, 1. ve 2. hatalarda servisin otomatik yeniden başlatılmasını sağlayın. Ayrıca SQL üzerindeki bağlantı havuzu (Connection Pool) ayarlarının dolup dolmadığını SQL Profiler ile izleyin.

Python ile BIST-30 Verisi Çekme ve Algoritmik Analiz

Finansal piyasa verilerini Python kütüphaneleri ile otomatik işleyerek arbitraj hesaplama.

Borsa İstanbul (BIST-30) verilerini analiz etmek ve otomatik işlem stratejileri (arbitraj) geliştirmek için Python kusursuz bir araçtır.

Web Mimarisi

B2B Platformları İçin React.js ve Node.js Mimarisi

Kurumsal müşterilere hitap eden hızlı, güvenli ve ölçeklenebilir bayi portalı tasarımı.

Modern B2B (Business to Business) platformları artık klasik MVC yapıları yerine, Frontend'de React.js, Backend'de ise Node.js (Express) kullanılarak inşa ediliyor.

Bu yapının en büyük avantajı, ERP sistemleri ile API üzerinden çok hızlı asenkron iletişim kurabilmesidir. JWT (JSON Web Token) ile sağlanan oturum yönetimi ve kurumsal yetkilendirme (Role-Based Access Control) sayesinde bayiler sadece kendi siparişlerini ve cari hesaplarını güvenle yönetebilir.

Performans

Büyük Veritabanlarında SQL Sorgu Optimizasyonu

Veritabanı şişmeleri, Indexing stratejileri ve Execution Plan analizi ile hızlandırma.

ERP sistemleri yavaşlıyorsa sorun genellikle donanım değil, yanlış yazılmış SQL sorguları veya eksik indekslerdir (Index).

Nereden Başlamalı?

SQL Server Management Studio'da Include Actual Execution Plan özelliğini açarak sorgunuzun nerede takıldığını (Table Scan vs Index Scan) görün. Sık WHERE ve JOIN yapılan kolonlara Non-Clustered Index atamak, saniyeler süren sorguları milisaniyelere düşürebilir.

Yazılım Ürünü

DomainMonitorX: Akıllı Takip Sistemi Geliştirme

Yüzlerce kurumsal domain ve lisansın bitiş tarihlerini otomatik takip eden yazılım projem.

IT yöneticilerinin en büyük kabusu, unutulan bir domain veya SSL sertifikası süresinin bitmesidir. Bu problemi çözmek için geliştirdiğim DomainMonitorX mimarisi, arka planda çalışan zamanlanmış görevler (Cron Jobs) ve Whois API'leri sayesinde bitiş tarihlerini önceden tespit eder.

Sistem, süre bitimine 30, 15 ve 3 gün kala IT ekibine otomatik e-posta, SMS veya Microsoft Teams üzerinden webhook ile bildirim göndererek kesintilerin önüne geçer.

Kategori 03

Elektronik & Endüstriyel Üretim

Devre Tasarımı

PCB Tasarımında Differential Pair (Diferansiyel Çift) Kullanımı

Altium Designer gibi araçlarda yüksek hızlı sinyal bütünlüğünü koruma kuralları.

USB, HDMI veya Ethernet gibi yüksek frekanslı sinyaller PCB üzerinde taşınırken Differential Pair (Diferansiyel Çift) tekniği kullanılır.

İki hat arasındaki sinyaller birbirinin tam tersi (faz farkı) olduğu için, dışarıdan gelen elektromanyetik gürültüler (EMI) her iki hata eşit etki eder. Alıcı entegre, bu iki sinyalin farkını hesapladığında gürültü matematiksel olarak sıfırlanmış olur. Çizim yaparken hatların aynı uzunlukta (Length Matching) olmasına çok dikkat edilmelidir.

Üretim & Tamir

Endüstriyel Elektronik Kartlarda Soğuk Lehim Çözümü

SMD bileşenlerde titreşim veya ısı farklılıkları kaynaklı oksitlenme sorunları.

Endüstriyel makinelerdeki elektronik kartlar yoğun titreşime ve yüksek ısı farklılıklarına maruz kalır. Bu durum zamanla Soğuk Lehim (Cold Solder) denilen mikro çatlakların oluşmasına sebep olur.

Çıplak gözle zor görülen bu arızalar makinenin aralıklı olarak durmasına neden olur. Çözüm için kartın mikroskop altında incelenmesi, şüpheli SMD bileşenlerin kaliteli flux (lehim pastası) ile yeniden Reflow istasyonlarında ısıtılarak lehim tazelemesi yapılması şarttır.

Gömülü Sistemler

ESP32 ve RS485 ile Modbus Okuma (Endüstri 4.0)

Fabrikadaki eski nesil PLC ve sensör verilerini Wi-Fi üzerinden buluta taşıma.

Sensör verilerini dijitalleştirmek (Endüstri 4.0) için eski makineleri değiştirmeye gerek yoktur. RS485 Modbus protokolü ile konuşan herhangi bir cihaz, uygun bir MAX485 entegresi ve ESP32 mikrokontrolcüsü kullanılarak akıllı hale getirilebilir.

ESP32, verileri Modbus üzerinden seri porttan okur ve içindeki dahili Wi-Fi modülü ile MQTT protokolü üzerinden merkezi sunucudaki bir veritabanına (Örn: InfluxDB) aktarır.

Donanım Projesi

Solar Sensörlü Endüstriyel Aydınlatma Ar-Ge Süreci

Güneş enerjisi ve lityum pil yönetim sistemleri (BMS) entegrasyon projemiz.

Dış mekan ve fabrika çevreleri için bağımsız çalışabilen solar aydınlatma sistemleri tasarlarken en kritik bileşen Pil Yönetim Sistemidir (BMS).

Solar panelden gelen değişken voltajı MPPT (Maximum Power Point Tracking) algoritması ile işleyerek lityum pilleri en verimli şekilde şarj eden bir devre tasarımı yaptık. Ayrıca PIR hareket sensörleri ile gereksiz enerji tüketimini önleyerek batarya ömrünü maksimuma çıkardık.
